In this post we see how to push files on GitHub using Git

So, first of all let's understand the Basic Concepts.

What is GitHub?

GitHub is distributed version control. Using GitHub you can frame your code. In future suppose you want your previous code file with the help of GitHub you can easily find. In GitHub they provide two sections public and private. The public part is free but you want to push your code in private. In that case you paid some amount .In the private section you can store your private code that you do not want to explore publicly. These kinds of things you can push to a private repository. Repository is nothing but the folder where you can store your data.

What is Git?

Git also has open-source distributed version control. Git is designed in the Unix base system that’s why it looks like a Linux command. With the help of Git you can push your code local directory to GitHub. Using some commands.

So, In this part we see some Commands.

Those helpful to do operations.

1. git status: the help of this command, you can check the current status of the directory.

2. git init: With the help of this command Initialize the git repository.

3. git add. :Add working directory to stage directory.

4.git commit -m “”: In Double quotes you can write any message/comment .That is helpful to understand you and another programmer also.

5.git log: Check the user log ,Understand what someone has changed.

6.git config --global user.name “Enter your name” : This command helps to set your name Globally ,means set this name for all repositories.

7.git config--global user.email “Enter your email” : This command helps to set your email Globally ,means set this email for all repositories.

Command for modified file to original file

git reset “file name”:reset the commit

2 git checkout “file name”:commit for file

3 git checkout . : old/previous Commit for all file

Staging basic in git

1 git add -A : Stages for all file

2 git add . : stages for new and modified files but without deleted files.

3 git add -u: stages for modified and deleted files but without new files.

Command for cloning the repository.

command : git clone “Enter SHA url”

Ignoring the file: Suppose in your project some file that you do not push on your repository that kinds of all files you can simply ignore. For example .exe and .c file in c programming language

1 touch .gitignore : Create a new black file

2 Notepad .gitignore : In this file write the file name that you want to ignore

Branching Suppose you change something or add something in your project but not change the original project. In that time you can create your own branch and add changes in your branch and then you can test the code and merge with the master branch.

1 git branch “branch name” : Create own branch

2 git checkout “branch name” : Switch to the branch

2.1: git add .

2.2: git commit -m “commit”

2.3: git push origin “your branch name”

Your new branch merge with master branch

1 git checkout master

2 git merge “your branch name”

3 git push -u origine master

Delete Branch Suppose you create a new branch and changes in that branch after merging these changes with master branch. so your branch work is done. They occupy extra memory, after merge with master there is no need for a new branch hence you can simply delete this branch.

1 git branch -d “branch name” : This command deletes your branch locally.

2 git push origin --delete “branch name”: This command deletesyour branch remotely.
